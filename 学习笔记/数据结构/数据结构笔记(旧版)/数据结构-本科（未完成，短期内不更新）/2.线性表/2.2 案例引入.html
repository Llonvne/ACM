案例2.1 一元多项式的运算，实现两个多项式加，减，乘，除运算

P_n(x) = p_0 + p_1x + … + p_ n x ^ n


把一元多项式的每一个系数拿出来，存成线性表


每一项的指数用系数的下标来表达


所以线性表 P = (p_0,p_1,p_3,…,p_n)

(每一项的指数i隐含在其系数p_i的序号中)


可以用数组来表示


这样我们就可以进行多项式的运算了


R_n(X) = P_n(X) + Q_m(X)


这个时候我们只需要将两个线性表系数对应相加


有时候会碰到一些稀疏多项式

S(x) = 1 + 3X^10000 + 2X^20000


如果也按照刚才的方法来存放就会十分浪费空间

我们可以在添加一个表，来存储下标

0 10000 20000

1 3 5

这样就可以少占用很多空间


线性表 P = ((p_1,e_1),….,(p_m,e_m))


那么稀疏线性表可以进行运算吗

可以，

  1. 我们需要创建一个新数组c，也是有指数和系数存储
  2. 分别从头遍历比较a和b的每一项
  • 指数相同：对应系数想加，如果和不为0，就添加到C中
  • 指数不相同：将系数较小的项复制到c中
  1. 一个多项式已经遍历完毕，将另外的剩余项依次复制到c中即可

数组c到底多大合适：应该是 线性表a + 线性表b 的大小（MAX） 还是最小？

顺序存储结构存在问题

  • 存储空间分配不灵话
  • 运算的空间复杂度很高

我们可以用链式存储结构




图书管理系统


需要的功能：

  1. 查找
  2. 插入
  3. 删除
  4. 修改
  5. 排序
  6. 计数

  • 图书表可以抽象为线性表
  • 表示每本图书抽象线性表中的元素

可以用顺序表，可以用链表


  • 选择适当的存储结构
  • 实现此存储结构上的基本操作
  • 利用基本操作完成功能

总结：

  • 线性表中的数据元素可以为简单类型，也可以为复杂类型
  • 许多实际应用问题所涉及的基本操作有很大的相似性，不应为每个具体应用单独编写一个程序
  • 从具体的应用中抽象出共性的逻辑结构和基本操作（数据结构类型），然后实现其存储结构和基本操作
