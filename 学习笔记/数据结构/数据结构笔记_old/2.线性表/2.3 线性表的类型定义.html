2.3 线性表的类型定义

#数据结构


抽象数据类型线性表的定义如下：


其中 list 是线性表的英文单词


ADT List{
	数据对象： D = {a_i | a_i 属于 Elemset ， (i = 1,2,3,...,n,(n >= 0))}
	数据关系： R = { < a_i-1,a_i > | }
}



基本操作:


线性表的初始化:

InitList(&L)
// 操作结果 : 构造一个新的空线性表


销毁线性表:

DestoryList(&L)
// 初始条件: 线性表 L 已经存在
// 操作结构: 销毁线性表 L


清除线性表

ClearLst(&L)
//初始条件: 线性表 L 已经存在
//操作结果: 将线性表 L 重置为空表


销毁内存表会把表直接从内存中清空,但是清空表本身还在


判断线性表是否为空

ListEmpty(L)
//初始条件 : 线性表 L 已经存在
//操作结果 : 如果线性表 L 为空表 则返回 TRUE 否则返回 FALSE


求线性表的长度

ListLength(L)
//初始条件: 线性表 L 已经存在
//操作结果: 返回线性表 L 中的数据元素的个数


获得元素

GetElem(L,i,&e)
//初始条件: 线性表 L 存在, 且 i >= 1 , i<= 长度
//操作结果: 用 e 返回线性表 L 中的第 i 个数据元素的值


查找元素

LocateElem(L,e,compare())
//初始条件: 线性表 L 已经存在, compare()是数据元素的判定函数
//操作结果: 返回 L 中第一个与 e 满足compare() 的数据元素的位序,若这样的数据元素不存在则返回值为 0


求元素的前趋

PriorElem(L,cur_e,&pre_e)
//初始条件: 线性表 L 已经存在
//操作结果: 若cur_e 是 L 的数据元素,且不是第一个,那么则用pre_e 返回他的前趋,否则操作失败,pre_e无意义


求元素的后继

NextElem(L,cur_e,&next_e)
//初始条件： 线性表 L 已经存在
// 操作结果：若有cur_e 是L 的数据元素，且不是最后一个，则用next_e返回他的后继元素
否则操作失败，next_e无意义


插入元素

ListInsert(&L,i,e)
// 初始条件： 线性表L已经存在 且i在范围 1<= i <= Length+1
// 操作结果： 在L的第i个位置前面插入新的元素e，L的长度加一


删除元素

ListDelete(&L,i,&e)
//初始条件： 线性表L已经存在，且 1 <= i <= length
//操作结果： 删除L中的第i个元素，并用e返回其值，L的长度减一


表遍历

ListTraverse(&L,visited())
//初始条件： 表L存在
//操作结果： 依次对线性表中的每个元素调用visited()


注意：以上提及的运算都是逻辑结构上定义的运算，只要给出这些运算的功能是做什么，至于如何做等实现细节，只有等待存储结构确定后才考虑
